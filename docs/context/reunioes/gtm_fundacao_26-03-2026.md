# GTM Fundação — AuroraISP

**Data:** 26/03/2026
**Participantes:** CEO, PMM (Claude)
**Duração:** Sessão longa

---

## Contexto

Primeira sessão de construção do GTM da AuroraISP. Partimos do zero, com base nos projetos existentes (`robo/` e `megaroleta/`) e em documentos de uma conversa anterior com IA. O objetivo foi construir a fundação estratégica do hub de tecnologia para provedores de internet.

---

## Principais pontos discutidos

- Definição da AuroraISP como hub de tecnologia com IA para provedores de internet
- Estrutura de 3 módulos: Comercial, Marketing e CS
- Marketing como motor de automação do hub (réguas padrão prontas)
- CRM Kanban como parte do Comercial Pro, não produto separado
- Clube de Benefícios como parte do módulo CS
- Modelo de precificação: mensalidade por módulo (Start/Pro) + transacional variável
- Criação de estrutura de agentes por função (13 agentes)
- Tom de voz: acessível, próximo, direto. Fala a língua do provedor.
- Regra de escrita: sem traço como pontuação
- Estrutura de docs em `docs/GTM/`, `docs/BRAND/`, `docs/AGENTES/`

---

## Decisões tomadas

| Decisão | Motivo |
|---------|--------|
| Nome do módulo de vendas: Comercial | Mais claro e direto que "Robô de Vendas" |
| CRM Kanban dentro do Comercial Pro | Depende dos dados do Comercial. Não faz sentido separado. |
| Clube de Benefícios dentro do CS | CS é mais amplo e completo do que só o clube |
| Marketing como motor de automação | Todas as réguas rodam via Marketing. CS e Comercial consomem os fluxos. |
| Tagline: Vende mais. Perde menos. Fideliza sempre. | Mapeia os 3 módulos sem precisar explicar |
| Subtítulo: Do lead ao cliente fidelizado com inteligência. | Complementa a tagline com a jornada e o diferencial de IA |
| Paleta oficial: dark mode (#0F172A, #2563EB, gradiente azul/ciano) | Extraída do site auroraisp.com.br (site do CEO) |
| Padrão de interface: light mode, dark automático via sistema | Site será light. App respeita preferência do sistema. |
| Transacional Marketing: tamanho da base de leads | Modelo recorrente que escala com o crescimento da base |
| Transacional Comercial: venda finalizada no ERP | Alinha receita da Aurora com conversão do provedor |
| Transacional CS: cliente ativo no clube | Modelo recorrente mensal |
