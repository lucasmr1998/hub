# Posicionamento por Jobs to be Done

Documentos de posicionamento organizados por **JTBD (Job to be Done)** — ou seja, pela **dor que o cliente quer resolver**, não pelo módulo/feature isolado.

---

## Ordem do funil

Cada JTBD cobre uma etapa da jornada do ISP crescendo:

```
JTBD-01 Marketing   ──(leads qualificados)──▶   JTBD-02 Comercial   ──(contratos)──▶   JTBD-03 CS
         ▲                                                                                    │
         └────────────────────────(ciclo via indicacoes)──────────────────────────────────────┘
```

## Template de cada doc

Cada JTBD responde 10 perguntas:

1. **Problema real** (a dor, não a feature)
2. **Quem sente** (persona por porte)
3. **Como resolviam antes** (status quo e concorrentes)
4. **Como o Hubtrix resolve** (solução tangível)
5. **Prova** (números e case)
6. **Pitch 30s** (pronto pra colar em cold call)
7. **Pitch 2min** (estrutura de demo)
8. **Objeções comuns + respostas**
9. **Como usar** (por função: SDR, demo, proposta)
10. **Dependências** (conexão com outros JTBDs)

---

## Como usar este material

### Para time comercial (SDR / Vendedor)

- Decore o pitch de 30s do JTBD mais relevante pro prospect
- Antecipe objeções (seção 8)
- Use o case do [30k assinantes](../cases/case_provedor_regional_30k.md) como âncora de prova

### Para demo

Cada JTBD tem um **roteiro de 2 minutos** (seção 7). Adapte pro contexto do prospect. **Não demo feature por feature** — demo por dor/JTBD.

### Para proposta

Cite:
- Os números do case (seção 5 de qualquer JTBD)
- ROI calculado com o volume específico do prospect
- Pitch conjunto dos 3 JTBDs (final do JTBD-03) pra stack completo

### Para marketing/copy

- Use os **ganchos** de cada JTBD (pergunta inicial do pitch 2min) como copy de:
  - E-mail outbound
  - Anúncio do LinkedIn
  - Primeira mensagem WhatsApp
  - Posts de blog/social
