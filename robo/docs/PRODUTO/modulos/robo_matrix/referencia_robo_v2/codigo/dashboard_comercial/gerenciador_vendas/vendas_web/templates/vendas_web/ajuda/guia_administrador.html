Guia do Administrador

Você enxerga e opera tudo. Este guia foca no que só o Administrador faz: decidir quem vê o quê, e configurar o comportamento do sistema.

Perfis de Acesso (Administração → Perfis de Acesso)

É aqui que você controla quem pode ver e operar o quê na ferramenta — sem precisar de deploy. Cada perfil é uma linha na lista à esquerda; ao clicar, você edita:

  • Capacidades "Ver" — quais páginas e pipelines aparecem no menu daquele perfil (Dashboard, Leads, cada um dos 5 pipelines, Tarefas, Desempenho, Retenção, Metas, Segmentos, Configurações, Admin).
  • Capacidades "Operar" — quais ações o perfil pode executar (mover card no kanban, editar lead, operar indicações, atribuir responsável, criar tarefas, gerenciar usuários/perfis/configurações).
  • Escopo de dados — Todos os registros (vê tudo), Registros dos pipelines que pode ver (vê tudo dentro do que está liberado) ou Apenas os atribuídos a ele (só os próprios).
  • Usuários — marque quem pertence a este perfil. Uma pessoa pode ter mais de um perfil ao mesmo tempo (as permissões se somam).
Como funciona por trás: a cada página carregada, o sistema calcula o conjunto de capacidades do usuário (união de todos os perfis dele) e usa isso para decidir o menu, as abas de pipeline visíveis e se cada botão de ação aparece habilitado. Tentar uma ação sem permissão (mesmo por URL direta) é bloqueado com erro 403.
Usuário sem perfil não tem acesso. Quem entra pelo portal sem um perfil atribuído vê a página "Seu acesso ainda não foi liberado" em qualquer tela. O acesso é liberado na hora em que você atribui um perfil (veja Gestão de Usuários abaixo). Administradores do portal continuam com acesso total.

Gestão de Usuários (Administração → Usuários)

Os usuários da ferramenta são os mesmos do Portal TecHub — ninguém cria senha aqui; o login acontece pelo portal (SSO). Nesta tela você:

  • Sincroniza os usuários do portal — o botão roxo importa/atualiza todas as contas do TecHub, para você poder liberar o acesso antes do primeiro login da pessoa.
  • Filtra por situação — Pendentes (sem perfil, bloqueados), Com acesso e Admins — e busca por nome, usuário ou e-mail.
  • Libera o acesso em um clique — botão "+ Liberar acesso" na linha do usuário → escolha o perfil. O × no chip remove o perfil (e o acesso, se for o único).
Fluxo recomendado: Sincronizar → filtro Pendentes → localizar a pessoa → + Liberar acesso → escolher o perfil. A mudança vale imediatamente, sem deploy e sem a pessoa precisar deslogar.

Mensagens do Robô (Administração → Mensagens do Robô)

Tudo que o robô escreve para o cliente no WhatsApp é personalizável aqui, sem deploy: perguntas da coleta (CPF, nome, endereço, fotos…), confirmações ("CPF validado!", "Anotado!"), mensagens de erro, as 3 mensagens de recontato (quando o cliente some) e a pergunta de retomada (continuar / recomeçar / outro CPF).

  • As alterações valem na hora — o robô recarrega os textos automaticamente.
  • Campo em branco = o robô volta ao texto padrão; nas confirmações, em branco significa não enviar nada (o robô segue direto para a próxima pergunta).
  • Não use emojis (podem virar "?" no WhatsApp do atendimento) — use *negrito*.
  • Mensagens com opções numeradas precisam manter os números (1, 2, 3…) — é assim que o robô entende a resposta do cliente.

Análises (Visão geral → Análises)

Hub único de métricas com filtro de período: funil de conversão, canal de Indicações (com ranking de indicadores), saúde da automação HubSoft (sucesso × falha, últimas falhas para investigar), operação do CRM (tempo por estágio, oportunidades paradas, carga por responsável) e o atrito do robô — em qual pergunta os clientes mais erram a resposta (ótimo insumo para melhorar os textos na aba Mensagens do Robô).

Mensagens de WhatsApp (Administração → Mensagens WhatsApp)

Cada um dos 5 pipelines tem uma mensagem inicial configurável. Ao clicar no botão verde de WhatsApp dentro de uma oportunidade, o app já abre com esse texto preenchido — o operador só confere e envia.

  • Use {primeiro_nome} e {nome} — são substituídos pelos dados reais do lead na hora de montar o link.
  • O campo ativa liga/desliga a mensagem sem apagar o texto salvo.

Config. CRM

Estágios de cada pipeline (nome, cor, ícone, ordem, se é estágio de ganho/perda), regras de movimentação automática do funil de Aquisição, tags e outras opções gerais do CRM ficam aqui. Mexer nos estágios de um pipeline atualiza automaticamente as abas do kanban e os relatórios — não precisa mexer em código.

Admin Django

Visão técnica/bruta de todos os dados: leads, prospectos convertidos, upgrades, novos serviços, execuções de automação no HubSoft (com status e erro de cada tentativa), atendimentos e questões do robô. Use para investigar um caso específico até o detalhe.

Contratos — o que o Administrador precisa saber

Nenhum operador (nem você) aceita contrato pelo cliente. O aceite acontece no app da Megalink, pelo próprio cliente. O CRM só monitora esse status (lido direto do HubSoft) e usa isso como trava: o atendimento e a O.S. de instalação só podem ser abertos depois do contrato aceito. Isso vale para todos os perfis, inclusive você.

Notificações personalizadas

O sino de cada usuário mostra só o que é relevante para o perfil dele: uma nova indicação avisa quem opera o pipeline de Indicações; uma falha de conversão ou de agendamento avisa quem tem acesso a Desempenho (gerentes/administradores); quando alguém atribui uma oportunidade a uma pessoa, só ela é avisada. Isso é automático — não exige configuração por evento, só depende dos perfis estarem corretos.

Por onde começar

  1. Confira os 5 perfis padrão em Perfis de Acesso e ajuste capacidades se necessário.
  2. Atribua cada usuário da equipe ao(s) perfil(is) certo(s).
  3. Revise as Mensagens de WhatsApp de cada pipeline.
  4. Leia os guias dos demais perfis (menu ao lado) para saber exatamente o que cada pessoa vai ver.
