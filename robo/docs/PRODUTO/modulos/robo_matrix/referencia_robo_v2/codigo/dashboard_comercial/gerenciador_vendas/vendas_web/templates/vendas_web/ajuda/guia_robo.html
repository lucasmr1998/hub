Como o Robô Atende no WhatsApp

Referência para todos os perfis: o que acontece nos bastidores quando o robô atende um cliente, e quando um humano precisa entrar.

A jornada no WhatsApp

Todo contato começa com o robô pedindo o CPF. A partir daí a conversa se divide:

Cliente manda "oi" Robô pede CPF É cliente? → Menu Não é? → Venda nova
  • Lead novo — o robô coleta nome, endereço (com CEP automático), plano, vencimento, fotos de documento e agenda a instalação. Tudo guiado, com validação por IA. Isso alimenta o pipeline de Aquisição no CRM.
  • Cliente existente — recebe o menu: 1) novo serviço · 2) upgrade de plano · 3) acompanhar instalação · 4) atendente · 5) finalizar. A opção upgrade só aparece se o cliente já tem algum serviço habilitado.
O robô não cadastra Indicações — esse pipeline é sempre iniciado por um Operador, manualmente, dentro do CRM (veja o Guia do Operador).
  • Cliente inativo/cancelado — se o CPF já teve cadastro na Megalink (mesmo cancelado), o robô reconhece e segue o fluxo de cliente existente: a pessoa pode contratar normalmente.
  • Cobertura (viabilidade) — ao confirmar o endereço, o robô checa se a cidade é atendida. Fora da área de cobertura (ou imóvel empresarial), a conversa é transferida para um atendente — o robô não segue vendendo.

Se o cliente para de responder (recontato)

Quando o cliente some no meio da conversa, o robô não encerra de primeira: ele tenta reengajar com até 3 mensagens diferentes e escalonadas (personalizadas com o nome), sem repetir a pergunta. Se mesmo assim não houver resposta, envia uma despedida e pausa em silêncio — o atendimento fica aguardando o cliente voltar.

Silêncio 3 mensagens de reengajamento Despedida (1x) Pausa aguardando
Se o cliente responder a pergunta pendente durante o recontato (ex.: mandar a data de nascimento), a resposta é aproveitada — o robô não pergunta de novo. Os textos dessas mensagens são editáveis em Mensagens do Robô.

Se o cliente volta no meio do cadastro (retomada)

Quem tinha um cadastro pela metade e manda "oi" de novo não cai no meio do formulário: o robô pergunta como a pessoa quer seguir:

  • 1) Continuar de onde paramos — retoma na pergunta pendente, com tudo salvo.
  • 2) Recomeçar do início — zera os dados (mantém o CPF) e recomeça do nome.
  • 3) É para outro CPF — zera tudo e recolhe o CPF (outra pessoa/cadastro).

Para clientes já cadastrados, a pergunta equivalente é o "é esse CPF ou outro?" antes do menu.

Upgrade de plano (opção 2)

O robô mostra os serviços ativos do cliente, os planos disponíveis (sem o atual), pede confirmação e registra a solicitação. Dali em diante:

Cliente confirma Solicitação registrada Robô executa no HubSoft (≤5 min) Migração aplicada ✓
A migração no HubSoft é feita por uma automação própria — API interna rápida, com o antigo processo manual como reserva se algo mudar do lado do HubSoft.

Novo serviço (opção 1)

Coleta guiada de endereço, plano, documentos e agendamento. Ao finalizar, acontecem duas automações em sequência:

  1. HubSoft — robô cria o novo serviço no cadastro do cliente (≤2 min), usando o endereço que o cliente informou para esse serviço (não o endereço cadastral).
  2. Matrix — o sistema abre o Atendimento + OS de instalação na data/turno escolhidos.

Onde acompanhar

  • CRM → abas Novo Serviço / Upgrade — cards andam sozinhos conforme o progresso (Coletando → Processando → Concluído/Falha).
  • Admin → Upgrades de Plano / Novos Serviços — visão técnica com status do HubSoft, OS do Matrix e mensagens de erro.
  • Notificações — o sino e a Central avisam os eventos importantes, para quem tem acesso àquele pipeline.

Quando o humano entra

  • Transbordo — cliente escolheu "Falar com Atendimento", o robô não conseguiu ajudar, ou não havia opções (ex.: upgrade sem serviço ativo). O lead fica marcado para atendimento humano.
  • Falha de automação — card em Falha no CRM ou status Falha no Admin: o robô tentou e não conseguiu aplicar no HubSoft. Faça o processo manualmente no HubSoft e registre nota na oportunidade.
  • Cliente "preso" — se uma conversa travar, o cliente pode mandar "oi" que o robô reapresenta o menu (fluxos abandonados são cancelados com segurança).
Nunca edite diretamente uma solicitação Pendente no Admin — o robô pode processá-la a qualquer momento. Em dúvida, fale com o gestor.

Notificações

Use a Central de Notificações (menu lateral) com o filtro Urgentes no início do turno — falhas de automação chegam para quem tem acesso a Desempenho (gerentes e administradores). Marque como lida ao tratar, assim o time sabe o que ainda está pendente.
