# Aurora como Primeiro Cliente — 31/03/2026

**Data:** 31/03/2026
**Participantes:** Lucas (CEO), Claude (Assistente)
**Tipo:** Decisão estratégica

---

## Decisão

A AuroraISP será a primeira cliente do próprio sistema. Isso valida o produto com uso real antes de vender para provedores externos.

---

## Escopo definido

### Fase imediata: Opção A (Suporte) + Opção C (CRM Interno)

**Suporte ao Cliente (pós-venda):**
- Sistema de tickets/chamados dentro do painel
- Provedor abre solicitação, acompanha status
- SLA por plano (Starter, Start, Pro)
- Base de conhecimento / FAQ
- Histórico de interações

**CRM Interno da Aurora:**
- Pipeline de vendas dos provedores (provedores como clientes da Aurora)
- Prospecção, qualificação, negociação, fechamento
- Acompanhamento de trials (14 dias)
- Métricas: MRR, churn, LTV, CAC
- Integração com os fluxos de atendimento já existentes

### Futuro: Opção B (Onboarding) + Opção D (Portal)
- Onboarding self-service
- Portal do cliente com billing, documentação

---

## Princípio

A Aurora usa o mesmo sistema que vende. Os módulos de Comercial e CS do próprio produto servem para gerenciar os clientes da Aurora. Isso garante que o produto funciona e que a equipe conhece cada detalhe.
