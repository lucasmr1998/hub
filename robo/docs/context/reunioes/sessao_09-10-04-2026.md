### 2. Fluxo FATEPI v2

**Implementado:**
- Novo fluxo completo (ID 9) com 18 nodos e 21 conexoes
- Estrutura: Nome → Curso → Validacao Curso → Ingresso → Criar Oport → Valores (IA) → Decisao → PIX → Matricula
- Agente IA centralizado com 3 tools (consultor_comercial, info_academica, suporte_geral)
- Fallback de perguntas → Agente IA → retorna para pergunta naturalmente
- Management command `criar_fluxo_fatepi` para recriar o fluxo

**Correcoes no engine:**
- Validacao falha com fallback conectado → redireciona para Agente IA
- Questoes sem IA seguem branch 'true' (corrigido de 'default')
- Agente retorna para pergunta com instrucao no prompt para retomar naturalmente
- `_validar_com_ia` corrigido para ler API key de `configuracoes_extras['api_key']`
- `ia_respondedor` agora e multi-turno (re-responde com historico)
- Questoes `espera_resposta=False` setam `_branch_saida` corretamente

**Simulador de Teste:**
- Botao "Testar" no editor de fluxos
- Modal lateral com chat: nome + telefone configuravel, reiniciar
- API `simular_fluxo_api` no backend

### 3. Filas de Atendimento e Transferencia para Humano

**Models:**
- `Conversa.modo_atendimento`: bot / humano / finalizado_bot
- `FilaInbox.mensagem_fora_horario`: mensagem automatica por fila
- `HorarioAtendimento.fila`: horarios por fila (FK nullable, null=global)

**Nodo "Transferir para Humano":**
- Tipo `transferir_humano` no editor de fluxos
- Config: fila de destino + mensagem de transferencia
- Engine: muda modo_atendimento, atribui fila/equipe, distribui agente, finaliza fluxo

**Distribuicao:**
- Agora por fila (removido global)
- Horarios verificados por fila
- Fora do horario: mensagem automatica + conversa pendente

**Visibilidade no Inbox:**
- Admin: abas Bot/Humano/Todas
- Supervisor: equipe + nao atribuidas da equipe
- Agente: suas conversas + nao atribuidas da fila
- Badge "Bot" visual na lista de conversas

### 4. Recontato Automatico

**Config por fluxo:**
- `FluxoAtendimento.recontato_ativo` + `recontato_config` (JSON)
- Tentativas configuraveis: tempo_minutos + mensagem
- Opcao IA para gerar mensagem contextual
- Acao final: abandonar ou transferir para humano

**Campos no AtendimentoFluxo:**
- `motivo_finalizacao`: completado, sem_resposta, abandonado_usuario, transferido, cancelado_atendente, cancelado_sistema, tempo_limite
- `recontato_tentativas`, `recontato_proximo_em`

**Cron:** `executar_recontato` (management command)
**Retomada:** Signal reseta contadores quando lead responde

**UI:** Botao "Recontato" no toolbar do editor, modal com config de tentativas

### 5. Modulo Suporte — Analise e Melhorias

**Correcoes de seguranca (7 fixes):**
1. Acoes de ticket restritas por permissao (`suporte.gerenciar_tickets`)
2. `tenant_cliente` fixo como `request.tenant`
3. `gerenciar_conhecimento` restrito por permissao
4. Atendentes filtrados por tenant
5. Paginacao em tickets (50/pag) e artigos (30/pag)
6. SLA breach calculado no banco (removido loop Python)
7. Delete de categoria protegido (bloqueia se tem artigos)

**8 features implementadas:**
1. **Respostas prontas** — dropdown no comentario, reaproveita RespostaRapida do inbox
2. **Historico/audit** — model HistoricoTicket com timeline na sidebar
3. **Auto-atribuicao** — CategoriaTicket.fila_padrao → selecionar_agente() do inbox
4. **Acoes em massa** — checkboxes + barra: mudar status, prioridade
5. **Alertas SLA** — cron `verificar_sla` detecta >80% e >100%
6. **Escalacao automatica** — SLA >150% eleva prioridade para urgente
7. **CSAT** — campos csat_nota/comentario no Ticket, widget de estrelas
8. **Relatorios avancados** — pagina com KPIs, por agente, categoria, prioridade, filtro periodo
