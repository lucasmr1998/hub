🎯 Visão Geral

O Megalink é um sistema completo de automação de vendas e gestão comercial desenvolvido para otimizar processos de conversão de leads em clientes. O sistema integra múltiplas tecnologias para fornecer uma solução robusta e escalável.

🎯 Objetivo Principal

Automatizar o processo de conversão de prospectos em clientes através de workflows inteligentes e integração com sistemas externos.

⚡ Performance

Processamento em tempo real com sistema de filas e notificações automáticas para máxima eficiência operacional.

🔒 Segurança

Implementação de autenticação robusta, controle de acesso granular e logs detalhados de auditoria.

📈 Escalabilidade

Arquitetura modular que permite expansão horizontal e integração com novos sistemas conforme necessário.

🏢 Sobre a Empresa

Megalink é uma empresa de tecnologia especializada em soluções de automação comercial e gestão de relacionamento com clientes. Nossa missão é transformar a forma como empresas gerenciam e convertem leads em clientes através de tecnologia avançada.

🎪 Principais Diferenciais

  • Automação Inteligente: Robôs que aprendem e se adaptam aos processos comerciais
  • Integração Total: Conecta todos os sistemas da empresa em uma única plataforma
  • Análise Preditiva: IA que prevê comportamentos e otimiza conversões
  • Escalabilidade Infinita: Suporta desde startups até grandes corporações
  • Suporte 24/7: Equipe técnica especializada sempre disponível

🤖 Sistema de Robôs de Automação

O coração da automação do Megalink é composto por robôs inteligentes desenvolvidos em Python com Selenium, responsáveis por automatizar completamente o processo de conversão de prospectos em clientes no sistema Hubsoft CRM.

📁 main_refatorado.py

Robô Principal - 784 linhas de código que executam a conversão automatizada de um prospecto específico através de 9 etapas bem definidas.

🔄 main_leads.py

Orquestrador - 457 linhas que gerenciam o loop infinito de processamento, buscando prospectos da API Hubsoft e executando o robô para cada um.

⏱️ Tempo de Processamento

Cada prospecto é processado em aproximadamente 45-60 segundos, com captura automática de screenshots em caso de erro.

🔁 Controle de Tentativas

Sistema inteligente com máximo de 3 tentativas por prospecto, evitando loops infinitos e marcando automaticamente como erro final.

📋 Fluxo Completo de Processamento (9 Etapas)

ETAPA 1 Login no Sistema Hubsoft
Acessa a URL de login, insere credenciais e valida acesso ao sistema CRM.
ETAPA 2 Navegação para Prospectos
Expande menu Cliente e navega até a seção de Prospectos.
ETAPA 3 Localização do Prospecto
Busca e filtra o prospecto específico por nome, maximizando a janela para visualizar todos os elementos.
ETAPA 4 Menu de Ações
Clica no botão "Ações" da linha do prospecto para abrir o menu de opções.
ETAPA 5 Converter em Cliente
Seleciona a opção "Converter em Cliente" e inicia o wizard de conversão.
ETAPA 6 Wizard - Primeira Tela
Preenche campos obrigatórios e seleciona opções no primeiro passo do wizard.
ETAPA 7 Wizard - Seleções Específicas
Configura parâmetros específicos (plano, dia de vencimento) no segundo passo.
ETAPA 8 Wizard - Terceira Tela
Avança para a tela final do wizard e seleciona configurações adicionais.
ETAPA 9 Finalização e Salvamento
Clica em SALVAR, confirma a conversão e atualiza status no banco de dados como CONCLUÍDO.

🔄 Classe Principal: ProspectoProcessor

conectar_banco()

Estabelece conexão com os dois bancos de dados (primário e Django) de forma redundante e tolerante a falhas.

desconectar_banco()

Encerra conexões de forma segura, garantindo que recursos sejam liberados corretamente.

salvar_prospecto()

Função CRÍTICA que salva status do processamento no banco primário e replica automaticamente para o Django.

capturar_screenshot_erro()

Captura screenshot automaticamente apenas em caso de erro, facilitando debug e análise de falhas.

⚙️ Características Técnicas

  • Modo Headless: Execução padrão em modo headless (sem interface gráfica) para otimização de recursos
  • Timeout Inteligente: WebDriverWait com 15 segundos de timeout para aguardar elementos
  • Scroll Automático: JavaScript para scroll até elementos antes de clicar
  • XPath Robustos: Uso de XPaths absolutos e relativos para localização precisa de elementos
  • Logging Detalhado: Prints com emojis para fácil identificação de etapas no console
  • Tratamento de Erros: Try-catch em cada etapa com rollback automático
  • Replicação Dual: Salva em dois bancos simultaneamente para redundância

🔁 Sistema de Controle de Tentativas

Primeira Tentativa

tentativas = 1
Primeira execução do robô. Se falhar, status fica como "erro" mas permite nova tentativa.

Segunda Tentativa

tentativas = 2
Segunda execução. Sistema ainda permite mais uma tentativa em caso de falha.

Terceira Tentativa

tentativas = 3
Última tentativa. Se falhar, marca como ERRO FINAL e não processa mais.

Bloqueio Automático

status = "erro final"
Prospecto é automaticamente pulado em futuras execuções do orquestrador.

📊 Mapeamento de Status Interno → Banco

PROCESSANDO Status Intermediários
INICIANDO, LOGIN_REALIZADO, NAVEGACAO_PROSPECTOS, PROSPECTO_LOCALIZADO, MENU_ACOES_ABERTO, WIZARD_INICIADO, WIZARD_TELA1, WIZARD_SELECOES, WIZARD_TELA2

🔄 Sistema de Replicação Automática

1. Salvamento Primário

Dados são SEMPRE salvos primeiro no banco primário com commit completo antes de qualquer replicação.

2. Replicação Django

Após commit no primário, tenta replicar para o Django. Falha não afeta o primário (já salvo).

3. Tolerância a Falhas

Se Django falhar, sistema continua funcionando normalmente. Logs registram erros de replicação.

4. Diferença de Status

Primário: "finalizado" | Django: "aguardando_validacao" (para revisão manual na interface)

📊 Sequência de Replicação

PASSO 1 INSERT/UPDATE no Banco Primário
Dados são inseridos ou atualizados no banco primário com todas as validações.
PASSO 2 COMMIT no Primário ✅
Commit garante que dados estão persistidos de forma permanente no primário.
PASSO 3 Verificar Existência no Django
Query para verificar se registro já existe no Django (por id_prospecto_hubsoft).
PASSO 4 INSERT/UPDATE no Django
Replica dados para Django com conversão de status (finalizado → aguardando_validacao).
PASSO 5 COMMIT no Django ✅ (ou Rollback se falhar)
Commit no Django. Se falhar, rollback no Django mas primário já está salvo!

📱 Sistema de Campanhas de Tráfego Pago

Sistema inteligente para rastreamento e atribuição automática de leads provenientes de campanhas de marketing digital. Detecta automaticamente a origem do lead através de palavras-chave em mensagens do WhatsApp, SMS ou outros canais.

Sistema Validado e Aprovado para Produção

20/20 testes aprovados (100%) - Performance < 100ms por requisição - Precisão de 100%

🎯 Visão Geral

📊 O que é?

Sistema que detecta automaticamente de qual campanha de marketing um lead veio, através da análise de mensagens recebidas via WhatsApp, SMS ou outros canais.

🎯 Como Funciona?

Quando um cliente menciona uma palavra-chave (ex: "CUPOM50"), o sistema identifica a campanha automaticamente e associa o lead à origem correta.

💡 Benefícios

  • ROI preciso por campanha
  • Taxa de conversão automática
  • Atribuição de origem correta
  • Análise de performance em tempo real

⚡ Integração N8N

Totalmente integrado via API REST para automatização completa do fluxo de detecção via webhooks do WhatsApp.

📝 Como Criar uma Campanha

Passos para criar campanha:
  1. Acesse: Dashboard → Campanhas de Tráfego → Nova Campanha
  2. Preencha os dados principais:
    • Nome: Ex: "Black Friday 2025"
    • Código: Ex: "BF2025" (único)
    • Plataforma: Google Ads, Facebook Ads, Instagram Ads, etc.
    • Investimento: Valor investido na campanha
  3. Configure a detecção:
    • Palavra-chave: Ex: "blackfriday" ou "black friday|bf 2025"
    • Tipo de Match: Exato, Parcial ou Regex
    • Case Sensitive: Marque se quiser diferenciar maiúsculas/minúsculas
    • Prioridade: 1-10 (quanto maior, mais prioridade)
  4. Defina o período:
    • Data de Início: Quando a campanha começa
    • Data de Fim: Quando a campanha termina
  5. Ative a campanha marcando "Ativa"

🎯 3 Métodos de Detecção (Matching)

1️⃣ Match Exato

Score: 100%

A palavra-chave deve aparecer exatamente como configurada na mensagem.

Exemplo:
Palavra-chave: PROMOCAO2025
✅ Detecta: "Vi a PROMOCAO2025"
❌ Não detecta: "Vi a promocao2025" (se case sensitive)

Quando usar: Para códigos promocionais específicos e únicos

2️⃣ Match Parcial

Score: 50-95%

A palavra-chave pode aparecer em qualquer parte da mensagem.

Exemplo:
Palavra-chave: cupom50
✅ Detecta: "Vi o cupom50 no Instagram"
✅ Detecta: "Quero usar cupom50"
❌ Não detecta: "cupom 50" (com espaço)

Quando usar: Para palavras-chave simples e diretas

3️⃣ Match Regex

Score: 90%

Usa expressões regulares para detectar variações da palavra-chave.

Exemplo:
Regex: black\s*friday|bf\s*2025|blackfriday
✅ Detecta: "Vi a blackfriday"
✅ Detecta: "Promoção Black Friday ainda vale?"
✅ Detecta: "Quero o desconto BF 2025"

Quando usar: Para campanhas com múltiplas variações de escrita

⚙️ Sistema de Priorização

Como funciona:

Quando uma mensagem contém múltiplas palavras-chave de diferentes campanhas, o sistema escolhe baseado em:
  1. Score mais alto (Match Exato 100% > Regex 90% > Parcial 50-95%)
  2. Se empate no score, escolhe a de maior prioridade
Exemplo:
Mensagem: "Vi a Black Friday e o cupom50, qual é melhor?"

Campanhas detectadas:
• Black Friday (Score: 90%, Prioridade: 10) ✅ VENCEDOR
• Cupom 50% OFF (Score: 65%, Prioridade: 8)

Resultado: Lead atribuído à campanha "Black Friday"

🔄 Integração com N8N

Fluxo completo de integração com WhatsApp via N8N:

Passo a Passo:
  1. Webhook do WhatsApp recebe mensagem do cliente
  2. N8N extrai telefone e mensagem
  3. N8N faz POST para /api/campanhas/detectar/
  4. Sistema detecta campanha e cria/atualiza lead
  5. N8N recebe response com dados da campanha
  6. N8N decide próxima ação baseado na resposta:
    • Se campanha detectada → Enviar mensagem específica da promoção
    • Se sem campanha → Enviar mensagem padrão de boas-vindas

📊 Analytics e Métricas

O sistema calcula automaticamente várias métricas para cada campanha:

💰 ROI (Return on Investment)

ROI = ((Receita - Investimento) / Investimento) × 100

Exemplo:
Investimento: R$ 1.000
Receita: R$ 5.000
ROI: 400% 📈

📈 Taxa de Conversão

Taxa = (Vendas / Detecções) × 100

Exemplo:
100 detecções
25 vendas
Taxa: 25% 🎯

💵 Receita Total

Soma de todas as vendas originadas da campanha, calculada automaticamente pelo sistema.

📊 Detecções

Número de vezes que a campanha foi detectada em mensagens de clientes.

🤖 Automação via Cron Job

Sistema de Processamento Automático (ATIVO)

O sistema possui automação completa para processar notificações pendentes a cada 5 minutos, 24/7, garantindo 100% de taxa de entrega.

📋 Configuração do Cron
Usuário: www-data
Frequência: A cada 5 minutos (288 execuções/dia)
Script: process_notifications.sh
Log: /var/log/notifications_cron.log
Comando Cron: */5 * * * *
Batch Size: 100 notificações por execução
Capacidade: Até 28.800 notificações/dia
Status: ✅ ATIVO
⚙️ Como Funciona
  1. Execução Automática: Cron executa script a cada 5 minutos
  2. Busca Pendentes: Procura notificações com status 'pendente' e data_agendamento <= agora
  3. Processamento: Envia até 100 notificações por vez via N8N
  4. Retry: Notificações falhadas são reprocessadas automaticamente (até 3 tentativas)
  5. Log: Registra todas as execuções em /var/log/notifications_cron.log
🔍 Comandos Úteis
# Ver cron configurado
sudo crontab -u www-data -l
# Ver última execução
tail -1 /var/log/notifications_cron.log
# Acompanhar em tempo real
tail -f /var/log/notifications_cron.log
# Processar manualmente
python manage.py process_notifications

  • Validação de Dados: Validação automática de emails, telefones e dados de contexto

  • Multi-Canal: Suporte a WhatsApp, Email, SMS, Push Notifications e Webhooks

🌐 Exemplo de Uso (JavaScript/API)

// Enviar notificação via API
fetch('/api/notificacoes/enviar/', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'X-CSRFToken': getCookie('csrftoken')
    },
    body: JSON.stringify({
        tipo: 'lead_novo',
        destinatarios: [1, 2, 3],
        dados_contexto: {
            lead: {
                nome: 'João Silva',
                empresa: 'Empresa ABC'
            }
        },
        prioridade: 'alta'
    })
})
.then(response => response.json())
.then(data => console.log(data));

// Obter estatísticas
fetch('/api/notificacoes/estatisticas/?data_inicio=2025-10-01&data_fim=2025-10-31')
.then(response => response.json())
.then(data => console.log(data.estatisticas));

  • Admin Interface: Gerenciamento completo via Django Admin
  • Perfil do Usuário: Interface dedicada para usuários editarem próprio telefone
  • API de Atualização: Endpoint RESTful para atualização via AJAX

🔗 Integração N8N

📤 Payload Enviado

JSON completo com notificacao_id, tipo, canal, destinatário, dados_contexto e timestamp

📥 Resposta Esperada

Status de sucesso/falha, execution_id, mensagem e timestamp de processamento

🔄 Workflow Recomendado

Webhook → Router → Canais Específicos → Response → Update Django

⚡ Performance

Timeout configurável (30s padrão), retry automático e processamento assíncrono

🔒 Segurança

  • Autenticação Obrigatória: Todas as APIs requerem login e token CSRF
  • Permissões Granulares: Superusuário, adm_all e usuários comuns
  • Rate Limiting: 100 requisições/minuto, 1000 notificações/hora
  • Validação de Dados: Sanitização de templates e dados de contexto
  • Logs Detalhados: Rastreamento completo de todas as operações

📈 Performance e Otimizações

  • Índices de Banco: status, data_criacao, tipo_id, canal_id
  • Processamento em Lotes: Máximo 100 notificações por vez
  • Cache de Templates: Templates renderizados são cacheados
  • Conexões Persistentes: Reutilização de conexões com N8N
  • Background Processing: Suporte a Celery para processamento assíncrono

Analisar erros comuns, validar dados de destinatários e implementar validação
