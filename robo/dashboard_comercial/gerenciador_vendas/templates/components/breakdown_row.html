{% load l10n %} {% comment %} Breakdown row — linha tabular pra dashboards de distribuicao (ex: por motivo de perda, por canal, por status). Sobre o `unlocalize` no width: o projeto roda em pt-br com USE_L10N, entao o Django renderiza 50.0 como "50,0". Dentro de style="width: ...%" isso e CSS invalido, o navegador descarta a regra e TODA barra aparece cheia. Falha calada: nao quebra a pagina, so mostra o numero errado. Como percentual quase sempre vem de um round(), o componente so acertava por acaso quando o valor dava inteiro. O `%` a direita continua localizado de proposito: ali virgula e o certo. Estrutura: label + meta (linha auxiliar) + barra de progresso colorida + pct destacado a direita. Params: label: titulo da linha (obrigatorio) meta: texto auxiliar abaixo do label (ex: "23 oportunidades — R$ 4.000") percent: numero 0-100 (largura da barra, e tambem mostrado a direita) variant: primary (default) | success | danger | warning | info — cor da barra pct_label: texto auxiliar abaixo do pct (default: "do total") bar_max_width: largura maxima da barra em px (default: 280) {% endcomment %}
